    Network Fundamentals Guide

    Learn how business networks work, one device at a time.

    What Is a Network Topology?

    A network topology is the arrangement of devices (nodes) and connections (links) in a computer network. In a business setting, the topology is designed for security, performance, and reliability. The topology shown in this lab follows a common hierarchical model used in small-to-medium businesses.

    Key Devices Explained

    🌎

    Edge Router

    Connects your business network to the Internet (ISP). It routes traffic between your internal network and the outside world using routing protocols and NAT.

    🛡️

    Firewall

    Acts as a security barrier. It inspects all incoming and outgoing traffic and blocks anything that doesn't match your security policies (rules).

    🔀

    Core Switch

    The high-speed backbone of your internal network. It connects all major segments together — servers, office floors, and other switches.

    ⚖️

    Load Balancer

    Distributes incoming requests (e.g. web traffic) across multiple servers so no single server gets overwhelmed. Ensures high availability.

    🔌

    Access Switch

    Provides network ports for end-user devices on each office floor. Supports VLANs to segment traffic (e.g. separating HR from Engineering).

    📡

    Wireless Access Point

    Provides Wi-Fi connectivity for laptops, phones, and tablets. Managed by a controller or configured individually with SSIDs and security.

    🖥️

    Servers

    Run business applications — web servers, databases, file shares, email. Typically placed in a server room behind the load balancer.

    💻

    Client PCs

    End-user workstations. They get IP addresses via DHCP, connect to file servers, and access the Internet through the network infrastructure.

    How Traffic Flows

    When a user on Floor 1 opens a website:

    1. PC sends request → Access Switch (Floor 1) 2. Access Switch → Core Switch 3. Core Switch → Firewall (inspects outbound traffic) 4. Firewall → Edge Router 5. Edge Router → Internet (ISP) 6. Response follows the reverse path back to the PC

    When accessing an internal web app:

    1. PC sends request → Access Switch → Core Switch 2. Core Switch → Load Balancer 3. Load Balancer → Web Server 1 or 2 (round-robin) 4. Server responds → Load Balancer → Core Switch → PC

    Essential CLI Command Modes (Cisco IOS)

    Most business network devices use a command-line interface (CLI). Here is how Cisco IOS modes work:

    Router> ← User EXEC mode (limited commands) Router> enable Router# ← Privileged EXEC mode (full access) Router# configure terminal Router(config)# ← Global Configuration mode Router(config)# interface g0/0 Router(config-if)# ← Interface Configuration mode Router(config-if)# exit Router(config)# exit Router# show running-config ← View current configuration

    VLANs — Why They Matter

    VLANs (Virtual LANs) let you segment one physical switch into multiple logical networks. This improves security (HR can't see Engineering traffic) and performance (less broadcast traffic).

    Switch(config)# vlan 10 Switch(config-vlan)# name ENGINEERING Switch(config)# vlan 20 Switch(config-vlan)# name HR Switch(config)# interface range fa0/1-12 Switch(config-if-range)# switchport mode access Switch(config-if-range)# switchport access vlan 10 Switch(config)# interface range fa0/13-24 Switch(config-if-range)# switchport mode access Switch(config-if-range)# switchport access vlan 20

    Subnetting Quick Reference

    Business networks typically use private IP ranges split into subnets:

    Network: 10.0.0.0/8 (Large enterprises) Network: 172.16.0.0/12 (Medium businesses) Network: 192.168.0.0/16 (Small businesses) Example subnets for this topology: Management VLAN: 192.168.1.0/24 (Switches, Router, Firewall) Servers: 192.168.10.0/24 Engineering: 192.168.20.0/24 HR / Admin: 192.168.30.0/24 Wi-Fi / Guest: 192.168.40.0/24

    Tips for Beginners

    1. Always save your config — Use write memory or copy running-config startup-config.

    2. Use "?" for help — Typing ? at any prompt shows available commands.

    3. Start from the edge — Configure the router and firewall first, then work inward to switches and endpoints.

    4. Test connectivity — Use ping and traceroute to verify your setup.

    5. Document everything — Keep a spreadsheet of IP addresses, VLANs, and port assignments.
